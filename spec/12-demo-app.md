## 12. Demo Application

### 12.1 Purpose and Scope
- The demo application is a separate Windows app for recording and comparing cursor visibility across Cursor Mirror, remote-viewer software, and other cursor-visibility tools.
- The demo application MUST move the real Windows cursor during Auto mode.
- The demo application MUST allow the user to enable or disable the demo-owned Cursor Mirror overlay before starting.
- The demo-owned Cursor Mirror overlay MUST be enabled by default.
- When the demo-owned Cursor Mirror overlay is enabled, the demo application MUST warn before starting if the main Cursor Mirror tray application is already running.
- The demo application SHOULD provide a path to request a normal shutdown of the main Cursor Mirror tray application before starting.
- The demo application MUST stop moving the cursor when the user takes control.
- The demo application is intended for demonstration capture and quick manual validation. It is not a replacement for the main tray application.

### 12.2 Application Shape
- The demo application MUST be built as `CursorMirror.Demo.exe`.
- The demo application MUST be included in the release package.
- The demo application MUST start as a normal visible Windows app.
- The startup view MUST provide a command to start the demo and a command to exit the application.
- When the demo is running, pressing any keyboard key MUST stop the demo and return to the startup view.
- Stopping the demo MUST unhook any demo-only hook and stop injecting cursor movement.

### 12.3 Startup Controls
- The startup view MUST allow selecting fullscreen mode or a window-size preset.
- The startup view MUST include at least `640 x 480`, `1280 x 720`, `1920 x 1080`, and fullscreen display modes.
- The default startup display mode MUST be `640 x 480`.
- The startup view MUST allow selecting the demo display language.
- The startup view MUST include a setting that controls whether the demo-owned mirrored cursor overlay is enabled.
- The startup view MUST expose the same user-facing movement translucency and predictive positioning controls as the main settings window.
- The startup view MUST expose idle-fade enablement, idle opacity, idle-fade duration, and idle-fade delay when the demo-owned mirrored cursor overlay is enabled.
- The startup view MUST disable mirrored-cursor parameter controls when their parent setting is disabled: prediction model, prediction gain, and prediction target offset when prediction is off, moving opacity/fade duration/idle delay when movement translucency is off, and idle opacity/idle-fade duration/idle-fade delay when idle fade is off.
- The startup view MUST disable all mirrored-cursor parameter controls when the demo-owned mirrored cursor overlay is disabled.
- The startup view MUST allow selecting one of three pointer speeds: slow, normal, and fast.
- The demo application MUST persist startup selections per user and restore them on the next launch.
- Demo settings writes MUST follow the durable settings write requirements in Section 5.6.
- Corrupt or unreadable demo settings MUST fall back to documented defaults without preventing startup.
- When demo settings restoration fails, the demo application MUST warn the user with a dialog and SHOULD reset the demo settings file to the defaults that are being used.
- Missing demo settings files MUST be treated as first-run defaults and MUST NOT show a restoration-failure warning.
- The startup view SHOULD keep explanatory text readable without clipping at the default dialog size.
- Disabled or unavailable controls SHOULD be avoided; when a control is present, it SHOULD be immediately meaningful for the next demo run.

### 12.4 Demo Scene
- The running demo MUST move the real Windows cursor horizontally inside a bounded region of the demo window.
- The demo scene background SHOULD be white or near-white so the real cursor and mirrored cursor are easy to inspect in recordings.
- When enabled, the demo scene MUST show a mirrored cursor overlay generated by Cursor Mirror's normal controller path.
- When disabled, the demo scene MUST NOT create the demo-owned mirrored cursor overlay, but MUST continue driving the real Windows cursor.
- Each Auto-mode run MUST start the deterministic cursor path at the left endpoint.
- The cursor path MUST include a stop at both horizontal endpoints.
- Endpoint entry and exit MUST use deterministic easing so the cursor starts and stops smoothly.
- The demo scene MUST display current status away from the cursor movement region.
- The status SHOULD include mode, speed, resume state, and injected movement count.
- The status MUST show whether the demo-owned mirrored cursor overlay is enabled as a separate top item above the other status fields.
- The status MUST NOT insert a blank line between the mirrored-cursor state and the next status field.
- The status MUST show the current real cursor X coordinate after mode, speed, and resume state as an integer relative to the demo path's left start endpoint, using a clear `X coordinate` label so separate recordings can be aligned.
- The demo scene MUST display the keyboard-stop hint in a separate panel from the status panel.

### 12.5 Real Cursor Driving and Free Mode
- Auto mode MUST move the real Windows cursor using injected mouse movement.
- Auto mode MUST use the shared `RealCursorDriver` SendInput path with the demo-specific injection marker.
- When the demo-owned mirrored cursor overlay is enabled, Auto mode MUST feed both demo-injected and user-generated mouse movement into Cursor Mirror's overlay controller so the mirrored cursor remains visible.
- Demo-injected mouse movement MUST carry a recognizable injection marker so the demo can distinguish its own movement from user movement.
- The demo MUST install a low-level mouse hook only while the demo scene is running.
- The demo hook MUST treat mouse movement, button, and wheel events without the demo injection marker as user control.
- When user control is detected, the demo MUST enter Free mode.
- Free mode MUST NOT inject cursor movement.
- If no user mouse input is detected for `3` seconds while in Free mode, the demo MUST return to Auto mode.
- Auto mode resumption MUST restart the deterministic path from the left endpoint instead of resuming from the previous elapsed path phase.
- The deterministic path generator MUST be testable without opening windows, installing hooks, or moving the real cursor.

### 12.6 Safety
- The demo application MUST NOT require administrator privileges.
- The demo application MUST NOT persist user settings unless that behavior is explicitly specified in a later section.
- Demo shutdown MUST dispose timers and demo-only hook resources.

### 12.7 Main Application Shutdown Request
- The main tray application SHOULD expose a per-session shutdown signal that can be requested by the demo application.
- A shutdown requested through that signal MUST use the same shutdown path as tray `Exit`.
- The demo application SHOULD only offer the main-application shutdown path when the demo-owned mirrored cursor overlay is enabled.
- The demo application MUST NOT forcefully terminate the main tray application when the shutdown signal is not available.
- If the shutdown request fails or cannot be confirmed, the demo application MUST keep the user on the startup view and explain that the tray application should be closed manually.

### 12.8 Testing
- Unit tests MUST cover deterministic path generation and Free mode timeout behavior.
- Normal automated tests MUST NOT install real Windows hooks or move the real cursor.
- Manual validation SHOULD cover startup controls, mirrored cursor enable and disable behavior, idle-fade settings, display-language selection, selection persistence after restart, non-clipped startup explanatory text, fullscreen mode, all window presets, keyboard-key demo exit, status visibility outside the cursor movement region, separate keyboard-stop hint visibility, mirrored cursor status visibility, Auto mode cursor movement from the left endpoint, Free mode entry, 3-second Auto mode resumption from the left endpoint, and release-package launch.
