# Cursor Prediction v5

v5 evaluates the first trace captured after adding the DWM-synchronized runtime scheduler stream to Cursor Mirror Trace Tool.

## Input Trace

- `../../cursor-mirror-trace-20260501-231621.zip`

## Key Change From v4

v4 treated the old product-equivalent WinForms timer `poll` stream as the runtime input proxy. v5 treats `runtimeSchedulerPoll` as the current runtime input proxy and keeps `referencePoll` as the dense reference stream for target reconstruction.

## Evaluation Policy

- Anchor stream: `runtimeSchedulerPoll`.
- Ground truth: linear interpolation of `referencePoll` at the target time.
- Product target: the same DWM next-vblank selection used by `DwmAwareCursorPositionPredictor`, recomputed from the runtime sample timestamp and DWM timing fields.
- Baseline: last-two runtime samples, gain `0.75`, idle reset `100ms`.
- Safety lens: a default-on candidate must improve tail error without adding meaningful visible regressions versus the baseline.

## Phase Layout

- `phase-1 runtime-scheduler-baseline-audit/`: data audit and baseline replay.
- `phase-2 deterministic-candidate-search/`: deterministic candidate search around the scheduler-backed runtime stream.

All phase outputs are generated by `analyze_v5.js`.
